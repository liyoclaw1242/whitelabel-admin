## Interpretation

The p95 breach is **driver-location artefact**, not a production issue:

- Driver (k6) runs on a laptop in Taipei
- DNS routes to Vercel iad1 (US East) functions
- Functions dial Neon ap-southeast-1 (Singapore)
- Each `/api/health` hop: Taipei → US → Singapore → US → Taipei ≈ 600 ms of speed-of-light RTT
- Vercel cold-start + chi middleware + pgx pool ping + OTel fan-out cost on top

Running k6 from a cloud region close to one of those hops (e.g. GitHub Actions runner in US East, or a sin1 VM) should show p95 closer to ~300-500 ms. The threshold was set assuming one such scenario and needs widening for ad-hoc local runs from APAC.
